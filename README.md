# Blinkit Sales Data Analysis (Power BI)

## 📌 Project Overview
This project analyzes Blinkit sales data to uncover trends, performance metrics, and actionable insights for business decision-making. It covers **data cleaning**, **key performance indicators (KPIs)**, and **detailed sales analysis** across multiple dimensions including Item Type, Fat Content, Outlet Type, and Outlet Location.  

The workflow includes **cleaning the raw data**, performing aggregations using SQL (stored in a separate query document), and visualizing the results using **Power BI dashboards**.

**Tools Used:**
- Power BI (visualization)
- SQL (data preprocessing and aggregation)
- Excel / CSV (dataset storage)

---

## 🧹 Data Cleaning
The dataset contained inconsistencies, such as multiple variations of `Item_Fat_Content` (e.g., `LF`, `low fat`, `reg`). Standardizing these values ensures consistent reporting and accurate insights.  

---

## 📊 Key Performance Indicators (KPIs)
- **Total Sales**  
- **Average Sales**  
- **Number of Orders**  
- **Average Rating**  

All KPIs are calculated from the cleaned dataset and used to inform the visualizations and insights.

---

## 📈 Key Insights
- Standardizing `Item_Fat_Content` ensures reliable aggregation and reporting.  
- “Regular” items generate higher sales across most outlet locations.  
- Larger outlets and key locations contribute disproportionately to total sales.  
- Pivot tables and segmented analysis help compare sales across outlets, product types, and categories effectively.
